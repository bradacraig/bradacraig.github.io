Problem Solving

Stuck on a problem...

I was stuck on the reverse loop challenge in the last sprint. I needed to make the console print on separate lines: “Three!” “Two!” “One!” “Blast Off!”. For some reason I couldn’t wrap my head around the correct way to write the loop so that it would loop backwards through the array and print each string to the console one by one.

Problem solving techniques

I started by trying what I thought might work just to see if I might get lucky, if not hopefully I’d get an error or see what I was able to get printing to the console.

Next I tried altering my code based on the results I got back from my failed first attempt.

After this I reached out to the tech-questions channel on discord as I was beginning to get confused by the error message and results I was getting. I got some good advice back which helped me to modify my loop. However, it still wasn’t working perfectly..

Finally I turned to chatgpt, explained what I was expecting to see and shared my code snippet and asked it to point out specifically where in the code I needed to look to change my input to receive the expected output. I use this as a last resort as often if the prompt isn’t worded correctly it will just give the answer away, diminishing my learning opportunity. In the end this helped me slot the final piece into the puzzle.

Reflecting on the problem

I felt frustrated and overwhelmed initially. I thought I’d wrapped my head around the basic concept of loops, and while this was true to some extent, I was easily tripped up by attempting to implement it in reverse. On top of this, the console errors didn’t seem very helpful and tended to confuse me more. Finally, throughout the whole process, as I was working with a loop it was very easy to crash the browser if I wrote the condition incorrectly, adding to the frustration.

Lessons learnt

  • I learned that trial and error is a valid way to solve a problem.
  • I learned that breaking the problem down into parts can help me focus on one piece at a time rather than letting the whole problem overwhelm me.
  • It’s better to ask for help sooner, rather than later.
  • Chat GPT is useful and very powerful but can sometimes give too much away in a learning environment unless given specific instruction.

An elegant solution

In page scrolling on my personal blog. I wanted to have a basic navigation bar where a link button called “BLOG” would scroll down to the section of the page where I listed my blog links. I knew I had to use an tag for the link but wasn’t sure the best way of linking within the same page.

I googled “In page linking HTML CSS” and read through a few examples. It seemed using an ID for the section I wanted to navigate to would be the best bet. I got it to work but it wasn’t perfect, it was snapping to the very top of the screen and was too abrupt.

Instead of trying to get everything working at once I focused in on creating some offset so that the section title wasn’t snapping to the top of the page. Again I did some googling here and implemented my findings by writing a css rule to add scroll-margin-top to the section.

Once I got that working I googled smooth scrolling as I had a hunch that was what I needed to use to stop it jumping so abruptly. I didn’t know how to write the css rule however so needed some context and examples before writing it into my css file.

Reflecting on the process

This felt much more empowering, I had a clear plan of what I wanted to achieve and was able to break the problem down into steps to avoid overwhelming myself. While I couldn’t exactly try random css rules to see if they worked, I had a fair idea of what to google and knew what I wanted to see as an outcome. I was glad I was able to achieve it on my own without reaching out for external help via discord or chat gpt.

Lessons learnt

I learned how to link and scroll to different sections within a webpage using html IDs and css rules for scroll-margin and smooth scrolling. In terms of problem solving, I learned to trust my intuition and give it a go.
